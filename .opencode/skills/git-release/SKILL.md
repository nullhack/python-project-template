## What I do
Manage the complete release process including version calculation, changelog generation, release creation, and themed naming based on commit/PR content analysis. Produces beautifully formatted GitHub releases.

## When to use me
Use this when ready to create a new release after features are complete and tested.

## Hybrid Versioning System

### Version Format
`v{major}.{minor}.{YYYYMMDD}`

**Components:**
- **Major**: Breaking changes (e.g., API changes, removed features)
- **Minor**: New features, significant enhancements, or same-day releases
- **Date**: Release date in YYYYMMDD format

### Version Bump Rules
```bash
# Feature release (minor bump)
v1.2.20260302 → v1.3.{today}

# Breaking change (major bump)
v1.2.20260302 → v2.0.{today}

# Same day release (increment minor by 2)
v1.2.20260302 → v1.3.20260302
```

## Release Naming Strategy

### AI-Generated Themed Naming

The release name is generated by analyzing the merged PRs and commits since the last release. Generate a unique adjective-animal pair that:
- Reflects the PR/commit content accurately
- Has not been used before in this project
- Is memorable and appropriate

### Guidelines for AI

**Good adjectives** (avoid generic ones):
- Emotional/sensory: `electric`, `radiant`, `fiery`, `velvet`, `crystalline`, `luminous`
- Movement: `surging`, `drifting`, `soaring`, `diving`, `spiral`
- Nature: `blooming`, `misty`, `aurora`, `tidal`, `verdant`
- Abstract: `boundless`, `infinite`, `hidden`, `silent`

**Good animals** (avoid overused ones like fox, owl, dolphin, cheetah):
- Exotic: `narwhal`, `axolotl`, `capybara`, `quokka`, `pangolin`, `kestrel`
- Aquatic: `jellyfish`, `seahorse`, `manta`, `cuttlefish`, `otter`, `gecko`
- Birds: `kingfisher`, `heron`, `ibis`, `stork`, `bird-of-paradise`, `kestrel`
- Insects: `firefly`, `butterfly`, `dragonfly`, `beetle`, `mantid`
- Mythical-inspired: `phoenix`, `griffin`, `pegasus`, `siren`, `chimera`

**Avoid repetition** - Check previous releases:
```bash
git tag -l --sort=-v:refname | head -10
```

## Release Process Workflow

### Step 1: Analyze Commits Since Last Release
```bash
# Get last tag
last_tag=$(git describe --tags --abbrev=0)

# Get commits since last release
git log ${last_tag}..HEAD --oneline

# Get merged PRs
gh pr list --state merged --limit 20 --json title,number,labels
```

### Step 2: Generate Release Name and Body

**AI Analysis Process:**

1. Read all commits and PRs since last release
2. Categorize changes: Features, Bug Fixes, Refactoring, Documentation, etc.
3. Identify dominant theme/vibe
4. Select adjective-animal pair that captures the essence
5. Write a brief explanation of why that name fits

### Step 3: Create Beautiful GitHub Release

The release body MUST follow this exact format:

```markdown
# Release {version} - {animal_name} {emoji}

> *"{poetic tagline}"*

## Changelog

### Features
- feat: description (#PR)

### Bug Fixes
- fix: description (#PR)

### Refactoring
- refactor: description (#PR)

### Documentation
- docs: description (#PR)

### Merges
- Merge pull request #XX from branch

## Summary

2-3 sentence summary of what this release accomplishes.

---
**SHA**: `{short_sha}`
```

### Example Release (v2.1.20260413):
```markdown
# Release v2.1.20260413 - Polished Gecko 🦎

> *"Like a gecko scaling a wall, this release polishes and adheres to better practices"*

## Changelog

### Bug Fixes
- fix: remove custom CodeQL workflow to resolve config conflict
- fix: reduce complexity in run function to pass CI checks

### Features
- feat: add workflow coordination skills and enhanced QA enforcement

## Summary

This release focuses on **polishing** the template with Docker cleanup,
CI compliance fixes, and enhanced QA enforcement skills.

---
** SHA**: `80be795`
```

## Step-by-Step Release Process

### 1. Analyze and Generate
```bash
# Get commits since last release
last_tag=$(git describe --tags --abbrev=0)
commits=$(git log ${last_tag}..HEAD --oneline)

# Get PRs
gh pr list --state merged --limit 20 --json title,number,labels

# Analyze and determine name/theme
```

### 2. Update pyproject.toml
```bash
# Calculate new version
current_date=$(date +%Y%m%d)
# Increment minor or major as appropriate
sed -i "s/version = \".*\"/version = \"2.3.${current_date}\"/" pyproject.toml
```

### 3. Update CHANGELOG.md
Add entry at top in format:
```markdown
## [v{version}] - {adjective_animal} - {YYYY-MM-DD}

### Changed/Added/Fixed
- description
```

### 4. Create GitHub Release
```bash
# Create the release with full formatted notes
gh release create v{version} \
  --title "Release v{version} - {Adjective Animal}" \
  --notes "$(cat <<'RELEASE_EOF'
# Release v{version} - {Adjective Animal} {emoji}

> *"{poetic tagline}"*

## Changelog

{categorized changelog}

## Summary

{summary}

---
**SHA**: `{short_sha}`
RELEASE_EOF
)"
```

## Hotfix Release Process

For same-day emergency fixes:
```bash
# Calculate minor increment
last_tag=$(git describe --tags --abbrev=0)
minor=$(echo $last_tag | cut -d. -f2)
date_str=$(date +%Y%m%d)
new_version="v$((minor + 1)).${date_str}"

# Quick hotfix release
gh release create $new_version \
  --title "Release $new_version - {Animal} (Hotfix)" \
  --notes "Emergency patch for {issue}"
```
